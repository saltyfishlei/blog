通过这篇文章，你可以学会：

  1. 搞懂 package.json 的几个重要字段
  2. 明白 import 引入模块时的机制，引入一个模块到底引入的是哪个文件

文件夹查找机制

  1. 如果加了“./“, “/“这样的具体路径， 那么就直接按照路径找
  2. 如果只写了模块名，就从node_modules中查找对应的文件夹

导入一个库

  1. 执行import ... from '...'
  2. 去查找该库是否有package.json文件，并且有对应的控制字段
  3. 如果有就去根据main等字段综合决定；没有就引入index.js

package.json核心字段

  1. main： 一般常用，浏览器和 node 环境均生效
  2. module : ESM 规范入口，node 无效
  3. browser： 浏览器环境下的入口
  4. exports ： 可以覆盖前面属性，定义 import 和 require 分别引入什么，node14.13 后支持

main && module && browser 的应用

在浏览器环境下使用 import 引入的字段读取优先级：browser > module > main ,都没有等于package.json（读取 index.js)

在 node 环境下使用 require 引入时读取main ， 其他无效, 如果没有 main 等同于没有package.json

exports 的应用

使用这种方式配置引入文件时，在组件中引入的库（文件夹）需要放在 node_modules 中才能生效
