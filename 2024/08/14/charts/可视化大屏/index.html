方案适配

可视化大屏的适配方案一般分为始终满屏适配与始终保持比例适配

始终满屏

这种方案一般用在屏幕变化不会特别的大项目，项目基本上不会用在比例非常不协调的大屏，设计图按 1920*1080 标准电脑屏幕比，所用的屏幕也基本在这个比例左右

实现思路

1．核心： 宽度使用vw，高度使用vh，严格按照设计图换算，并且利用 flex 布局留出缓冲空间。

2．具体算法，如果设计图为 1920*1080：

高度上：

1. 图表高度/ 设计图整体高度 （ 某个值/1080）

  1. 得到的结果 *100,

        3. 计算结果加上`vh`单位

宽度上：

换成对应的宽度即可，与高度计算同理。

tips1 :

特别提醒： 因为大屏项目的高度始终要保持一屏， 一般移动端项目不用一屏，是可以滚动的，所以移动端高宽统一用vw

tips2:

大屏可视化获取数据的时候在需要对数据进行复杂操作时， 建议一次性将所有数据拿回来进行处理，然后再传递给展示数据的组件

如果后端返回的数据按照图表进行返回的，那么就可以在对应的图表组件中进行接口处理

始终保持比例适配

核心：用按比例用 scale 适配

  1. 开发项目直接使用设计图量出的尺寸填写大小
  2. 页面打开，读取当前屏幕宽高
  3. 按当前屏幕宽高，通过 scale 缩放整个项目
    1. 屏幕宽高比大于设计图，则根据高度进行缩放
  4. 让项目高度，横向竖向始终居中

扩展： 地图处理

用 echarts 绘制地图的三种方案

  1. 使用 map 图表

    用json数注册为地图。就可以画出一片地图

  2. 使用 geo 坐标系

    地图注册后不直接绘图 ，而是作为坐标系

  3. 直接使用百度地图，高德地图

    直接用百度地图或者高德地图作为坐标系，不用自己注册地图

前两种方法都需要注册地图到 Echarts 库上

map 地图

这种方式需要自己处理 json 数据，搜索阿里云数据可视化 datav

使用场景：通常来说，如果单纯的要可视化的说明每个地区的数据，用这种方案即可

1,地图也有 label，legend 等的调节, 和其他图表一样，用itemStyle可以调节地图每个区域的几何图形的样式

2，相对于其他图表，地图特有的属性有

  • 数据映射-nameMap
  • 缩放相关-scaleLimit，zoom

基础配置

visualMap 组件

visualMap一般配合热力图或者地图，根据数值显示不同的颜色

1，inRange 调控颜色变化范围

2，min，max 调控最大最小值

3，left 调节组件位置

4，text 调节组件最高最低文本

geo 地理坐标系绘制地图

如果我们以地图为基础，在地图的某个位置绘制一些东西，这个时候可以用 geo 地理坐标系来绘图

1，调节基本上 map 图表一致

2，如果需要对某一个区域特殊配置，要使用regions，而 map 里是在 data 里写

搭配 lines 绘制路线

lines是一种图表类型，我们经常使用 geo 搭配 lines 绘制路线图

1,lines 线条受 lineStyle 调节

2, 区别于 line 折线图，lines 主要用来绘制多个线条，数据的写法也不一样，而且 line 是不用用于 geo 坐标系的

百度/高德地图

百度高德地图等于直接引入百度地图作为你的 geo 坐标系，可以让地图更好看。也不用引入 json 了

1,需要引入百度地图 js，百度地图内置支持，高德地图需要引入对应插件

2,使用 bmap 配置，配置项和 geo 差不多
