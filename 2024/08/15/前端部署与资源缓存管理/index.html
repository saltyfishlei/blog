缓存怎么工作

我们需要知道浏览器对于我们加载过的图片，js，css 等等文件资源都会有缓存。首先我们明确缓存分为两种

  1. 强缓存
  2. 协商缓存

浏览器如何确实是哪种缓存? —— 最常见的是响应头cache-control

  • public,max-age=0 或者no-cache都是协商缓存
  • public,max-age=xxx 并且时间没过期，为强缓存
  • no-store为不缓存

强缓存与协商缓存的区别？

  • 强缓存
    • 资源只要设定的时间没有过期，文件 hash 没有变化，那么就会使用缓存资源， 状态码为灰色 200
  • 协商缓存
    • 每次都会请求核对服务器资源文件的最后修改时间是否或者etag与本地一致, 一致则用缓存的，状态码为 304

实践

当我们为协商缓存时:

资源文件内容完全不变，但是重新打包，文件修改时间发生变化，那么就会重新去请求资源(状态码为 200)，后续都是 304 状态

tip: 替换静态资源不用重启服务器

当我们为强缓存时:

资源文件修改时间发生变化，仍然使用强缓存资源（除了被请求的资源，比如 html)，并且状态码为灰色 200

但是如果 JS 文件 hash 名改变了，那么就会重新请求 JS 文件， css 文件仍然是使用缓存

优缺点

协商缓存：

​ 优点:能够保证每次前端打包后丢上服务器资源一定更新
​ 缺点:只要文件是新放的，即使文件内容没变也不缓存

强缓存:

​ 优点: 一定期限内，根本不用向服务器询问，一定是拿到缓存。速度最快
​ 缺点: 如何不配合 hash，无法感知到文件更新

hash 值最佳策略

  1. 每个文件有自己的 hash 值，确保不会被其他文件的改动影响
  2. 某个文件变更尽量不要影响跟他无关的文件 hash 值
