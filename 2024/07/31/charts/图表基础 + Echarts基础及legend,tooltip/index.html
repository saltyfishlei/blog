图表的本质： 先建立一个坐标系，然后把数据转化到坐标系中的

图表基础单词速记

不同图表的相同元素都是使用的相同的单词，记住对应的单词就能触类旁通。

legend**: 图例

xAxis： X 轴

  1. label - 轴上的文字

    1. line - 轴线
    2. tick - 刻度
    3. name - 名字

YAxis： Y 轴

tooltip: （鼠标悬停某点的）提示

line：（折线）图的线

图表分类

折线图

通常叫line ，通常用于展示走势

柱状图

通常叫bar, 也常用于展示走势

饼图

通常叫pie, 常用于展示占比

其他

散点图scatter, 雷达图radar, 漏斗图-funnel， 热力图-heatmap, 水波图-liquid, 进度条-progress，地图-map，旭日图-sunburst等等

Echarts

优点：简单易上手，图表功能全

缺点：配置项处理，不利于高度自定义的图表

基础知识

初始化

绘制图表都需要遵循的步骤

  1. 写一个 div 或者 canvas 元素作为容器
  2. 准备好配置
  3. 准备并转化数据
  4. 渲染

图表大小如何决定

如果 init 方法中有在第三个参数配置，那就按 init 的来，否则读取挂载 dom 的宽高。最终仍然要配合设备像素比决定最终宽高。

设备像素比（Device Pixel Ratio，简称 DPR 或 dpr），是描述屏幕物理像素与逻辑像素之间关系的一个比率，在早期的低分辨率屏幕上，物理像素和逻辑像素通常是 1:1 的关系

init 方法的其他次要配置项

  1. renderer- 渲染模式，可以使用 svg 或者 canvas
  2. useDirtyRect- 脏矩形渲染，部分更新，用来提升性能
  3. useCoarsePointer和pointerSize- 扩大点击范围
  4. ssr - 是否服务端渲染，开启后不会自动渲染

配置项

哪些配置项是必要的？

  1. series-数组，用于定义画哪些图表，图表的数据，图表是什么类型（必填）
  2. xAxis- x 轴配置，定义 x 轴的取值和类型（如果是直角坐标系的图形必填）
  3. yAxis-y 轴配置，定义 y 轴的取值和类型（如果是直角坐标系的图形必填）

绘制

关于setOption:

  1. 如果要重绘图表，只需要传入部分配置，不传第二个参数，会进行合并
  2. 如果只想保留传入的配置，那就将第二个参数（notMerge）设为true
  3. 重绘时，如果配置有 id，那么就会去覆盖相同 id 的数据，否则就会新增数据

无论是 Vue 还是 React，组件的更新都是不会触发 echart 的更新（就算是响应式数据）， 我们需要主动调用setOption方法更新图表

item 图例的样式调节

  1. 图形的大小与间距: itemWidth, itemHeight, itemGap
  2. 图形的形状，自定义图形，使用图片作为 item： icon

  4. 线条的调节

  2. 文字的内容的自定义，基本上都会叫做formatter， 分为字符串与方法两种写法

图例整体位置和样式调节

  1. 基本上都会有 width 和 height 调节大小
  2. 有 border 系列调节边框，shadow 系列增加阴影
  3. Left，right，top，bottom，position，offset 等调节位置，不同的组件可能用的不同
  4. 颜色等相关样式，都会有 color，backgroundColor 等字段

legend 的width最大长度实际上是 itemWidth+itemHeight + itemGap , 最小宽度实际上也要保证一个 itemWidth

图例的一些特殊需要

  1. 竖向排列图例

    一定要注意图例的宽度与间距，不能超出图表高度，否则无法实现效果

  2. 图例太多可滚动

    设置type为scroll（最好设置高度height）

tooltip

主要分为content与axisPointer

content 的本质

  • content 部分默认是用 html 去绘制的，使用一定要使用 div 作为整个图表的容器
  • 如果使用 canvas 去绘制，必须使用 canvas 模式

为什么建议用 html 模式去绘制？ —— 可以直接使用 css 调节样式

触发设置

一般图表里的行为触发都叫 trigger

  • 触发方式-triggerOn
  • 触发位置-trigger

tooltip 样式和位置修改

  • className 调节（必须是 html 模式），如果是 canvas 模式，就得用富文本来调节了
  • 也有backgroundColor，textStyle，border，borderColor等基础配置调节
