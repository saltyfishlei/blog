性能优化

React 最大的一个性能问题是 —— 某个组件的 state 或 props 变化会触发整个函数的重新执行并且连带着他的子组件一起更新

解决办法： React.memo , 本质上是个高阶组件（类组件是PureComponent)

使用场景一般是props 不经常更新而且组件又存在较大的渲染开销

props

如果组件已经使用了React.memo 或者PureComponent来实现如果父组件传递的 props 没改变，就不会更新。但是我们需要注意父组件传入的方法，对象，数组这样的引用类型。

解决办法：

  1. 用useCallBack包裹传递给子组件的方法
  2. 非 state 对象，数组数据，要用useMemo包裹起来

tips： 不是只要传给子组件的对象都要用 useMemo 包括起来，固定数据我们可以写在函数体外面；使用 useMemo 一般是需要根据 state 计算出某种状态的情况。
