条件渲染

渲染原则：

  1. react 渲染 undefined, null, 空字符串，false 风不会渲染任何内容
  2. 如果渲染一个 jsx 编写的 html 元素，就会渲染成页面上的内容

所以我们就可以通过编写一个 JS 逻辑运算， 通过返回不同的值来决定是否够渲染该内容；

列表渲染

  2. 那么我们编写一个里面存放的都是 html 结构的数组，就会渲染成列表

生命周期

仅存在于类组件中，分为挂载，更新，卸载三大阶段

constructor: 初始化值

render: 决定要渲染的 dom，初始化及每次更新时触发

componentDidMount : DOM 挂载完成时触发

shouldComponentUpdate：更新时触发， 决定是否继续组件的更新（内置于PureComponent)

componentDidUpdate： 组件状态更新时触发, 如果shouldComponentUpdate返回 false 不执行；

componentDidUnmount： 组件卸载时触发

初始化顺序 ：constructor —> render —> componentDidMount

更新顺序： shouldComponentUpdate —> render —> componentDidUpdate

ref

用于获取真实 dom，与 vue 中的 ref 类似

注意：

  1. ref 必须在挂载后才能获取
  2. 不能获取函数组件（因为没有实例）

context

类似于 vue 的 provider 和 injected， 用于组件深层次的向下传值。

我们需要做三步：

  1. 创建一个context

  2. 从父组件传递指定的数据

  3. 在需要该数据的地方使用该 context
