函数组件

基本概念

前面的示例代码都是类组件，现在开始学习目前开发主流方式的函数组件；

与类组件的区别：

  1. 函数组件没有生命周期
  2. 函数组件没有 this
  3. 函数组件通过 hooks 来完成各种操作
  4. 函数体相当于 render 函数
  5. props 在函数的第一个参数接收

useState

参考 ： https://react.dev/learn/state-a-components-memory

useEffect

主要用于生命周期的模拟与监听等；

不传第二个参数 等于 componentDidMount和componentDidUpdate

第二个参数传空数组 等于 componentDidMount

第二个参数非传空数组 可以起到watch监听

所以 callback 初始化时始终都会调用一次；

useMemo

可以缓存渲染的计算结果，后续依赖的数据发生变化时才会重新运算；

类似 vue 的computed

useCallback

缓存一个方法，让方法不会每次更新重新创建

案例： 配合memo实现子组件的非必要不更新

tips： 上面提到的三个 hooks 的第二个参数用法基本一致，但是 useCallback 的应用主要是针对缓存机制，毕竟组件更新也不会触发函数。

useRef

主要用于在函数组件中保存和访问 DOM 节点或组件实例

useContext

读取并订阅从父级组件传递的value

高阶组件（HOC）

高阶组件（Higher Order Component，HOC）本质上是一种函数，并不是一种新的组件开发方式。

它接受一个组件作为参数, 并返回一个新的增强版组件

高阶组件可以用于在不改变原始组件代码的情况下，添加额外的功能或逻辑。
